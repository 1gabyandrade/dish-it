# Dish-It: Founder's Manual

Dish-It is a Streamlit web app that helps users turn the ingredients they
already have into simple, realistic recipes.

The product is built for students and young professionals who want to waste
less food, spend less money, and decide what to cook faster.

## What It Does

Dish-It lets a user create an account, select ingredients from an autocomplete
list, generate a recipe with AI, and save useful recipes for later.

Current features:

- Account creation and login
- Logged-in navigation inside the current browser session
- URLs do not carry login tokens that can be reused in another browser
- Ingredient search and selection
- AI recipe generation through OpenRouter
- Favorite recipes saved in `My Recipes`
- Recipe generation history saved in `History`
- Editable account page with username, email, and password updates
- Account deletion with a confirmation step
- SQLite database storage

Planned feature:

- `My Pantry`, where users will eventually manage the ingredients they usually
  keep at home

## Product Flow

1. A user signs up or logs in.
2. The user selects at least two ingredients.
3. Dish-It asks the AI service to create one practical recipe.
4. The generated recipe is shown on the home page.
5. The recipe is automatically added to `History`.
6. The user can save it to `My Recipes`.
7. The user can manage account details from `My Account`.

## Tech Stack

- Python
- Streamlit
- SQLite
- OpenRouter API
- `streamlit-searchbox` for ingredient autocomplete
- `bcrypt` for password hashing

## Requirements

Use Python 3.10 or newer.

Install dependencies from `requirements.txt`:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Environment Variables

Dish-It uses OpenRouter for recipe generation.

Create a `.env` file in the project root:

```bash
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

If the API key is missing or the API request fails, the app will show a friendly
"no recipe found" message instead of crashing.

## Run Locally

From the project root:

```bash
streamlit run app.py
```

Streamlit will print a local URL in the terminal, usually:

```text
http://localhost:8501
```

Open that URL in your browser.

## Database

The app uses SQLite.

The database is initialized automatically when the app starts. The schema lives
in:

```text
database/schema.sql
```

The app creates these tables:

- `users`
- `favorite_recipes`
- `recipe_history`

The local database file is created at:

```text
database/dish_it.db
```

## Main Pages

### Home

The user selects ingredients and generates a recipe. Generated recipes can be
saved to `My Recipes`.

### My Recipes

Shows recipes the user explicitly saved.

### My Pantry

Placeholder page for a future pantry-management feature.

### History

Shows all valid recipes generated by the user.

### My Account

Lets the user update username, email, and password. Password changes require the
current password plus the new password entered twice. The user can also delete
the account after confirming the action.

## Troubleshooting

If imports fail after adding a new service file, stop and restart Streamlit:

```bash
streamlit run app.py
```

If recipe generation does not work, check:

- `.env` exists in the project root
- `OPENROUTER_API_KEY` is set correctly
- the virtual environment is active
- dependencies are installed

If the UI looks outdated after CSS changes, refresh the browser tab.

## Founder Notes

Dish-It is currently an MVP. The focus is not to support every cooking case, but
to prove that people can quickly move from "what do I have?" to "what can I
cook?" with very little effort.

For a record of technical hurdles and fixes, see:

```text
TRIAGE_LOG.md
```

The strongest next product improvements are:

- Build the real `My Pantry` feature
- Add recipe filters such as time, difficulty, and dietary preference
- Let users remove items from history
- Improve generated recipe quality with stricter prompt rules
- Add tests for account, favorites, and history flows
